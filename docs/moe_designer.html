📖 MoE Studio Architecture Guide

WebGL Visualization Mapping ("The Balls & Beams")

This developer dashboard provides a real-time, interactive simulation of our Mixture of Experts (MoE) neural network architecture running locally on device.

  • 🟣 Central E8 Lattice Core: The complex wireframe structure in the center of the 3D space. It represents our Gemma-4-E2B Gateway Router gating network. Prompt embeddings are classified by projecting them into an 8-dimensional space derived from the 240 root coordinates of the $E_8$ Lie group lattice.
  • 🟢 Orbiting Expert Spheres ("The Balls"): The specialized colored spheres orbiting the central core represent individual expert models (e.g. CodeLlama, MathLLaMA). They wait in idle orbit until the gating network router dispatches tokens to them.
  • 🔵 Concentric Wavefront Rings: Flat concentric rings on the X-Z plane around the central E8 core. When a routing event occurs, these rings flash and scale up to represent a pulse wave, and the gateway router label dynamically changes from "GATEWAY" to "ROUTING".
  • ⚪ Outward Flow Particle Beams: Glowing streams of particles traveling outward from the central gateway along the connection lines to active expert spheres. This demonstrates the exact direction of data flow from the gateway router to the target experts.
  • 🔥 Rising Orange Embers (Ambient Heat): Orange particles rising from expert spheres when they are selected, showing that the expert is actively computing.
  • 📊 VU Level Meters & Expert Matrix: Located on the right sidebar, these track real-time routing weights (Top-K marginal probabilities) and throughput per expert.

Interactive Sandbox Features

  • Auto-Pilot Demo: Periodically dispatches automated mock queries to show dynamic routing transitions.
  • Simulate Query: Triggers a single query simulation.
  • Compile & Graft Expert (JIT Distiller): Compile new expert configurations in real-time, which then dynamically orbit the core.
  • Model Config Ingestion: Drag and drop custom Safetensor/GGUF JSON configs directly onto the vortex to load new models.
