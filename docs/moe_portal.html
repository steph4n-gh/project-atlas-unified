📖 Guide: 8-Expert Swapping Mechanics [Doc]

How the Mixture of Experts (MoE) Router works:

  1. Gateway Routing: When you submit a prompt, the Gateway Router (Class 7) uses vocabulary mapping and MLX logits to determine which expert domain is best suited (e.g., SQL queries to database_sql, neural math to ml_tensors).
  2. Dynamic Active-Path Swapping: To keep VRAM usage strictly bounded under 4GB, cold weights are evicted. The custom Weight Manager memory-maps (mmap) only the required expert model (uce_expert.safetensors) and pages active branches into VRAM on-demand.
  3. Sub-4ms Swap Latency: Because weights are cached on SSD and mapped via disk pages, swapping from one active expert path to another is completed in under 4ms.
  4. Autoregressive Generation: Once paged, UCE executes sparse active-path inference to generate domain-specific tokens.
